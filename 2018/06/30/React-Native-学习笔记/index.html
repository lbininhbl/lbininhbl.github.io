4 State（状态）

我们使用两种数据来控制一个组件：props 和 state 。props 是在父组件中指定，而且一经指定，在被指定的组件的生命周期中则不再改变。对于需要改变的数据，我们需要使用 state 。

一般来说，需要在 constructor 中初始化 state ，然后在需要修改时调用 setState 方法。

具体使用直接参考文档中的例子。

5 样式

RN 中的样式还是使用 JavaScript 来写，所有的核心组件都接受 style 属性。不过命名则为驼峰式命名，例如 CSS 上 background-color 则在 RN 里为 backgroundColor 。

样式的继承是按顺序渲染，相同名字的属性，后声明的属性值覆盖先声明的属性。

6 高度与宽度

最简单的给组件设定尺寸的方式就是在样式中指定固定的 width 和 height 。React Native 中的尺寸都是无单位的，表示的是与设备像素密度无关的逻辑像素点。

组件样式中使用 flex 可以使其在可利用的空间中动态地扩张或收缩。flex: 1 可指定某个组件扩张撑满剩余的空间。但是组件能撑满剩余空间的前提是父容器的尺寸不为零。

7 使用Flexbox布局

Flex Direction

在组件的 style 中指定 flexDirection 可以决定布局的主轴，默认是子元素沿着竖直轴（column）方向排列，可以设置水平轴（row）方向排列。

Justfy Content

在组件的 style 中指定 justifyContent 可以决定子元素沿着主轴的排列方式。可选项有：flext-start 、center、flex-end、space-around 以及 space-between 。

Align Items

在组件的 style 中指定 alignItems 可以决定其子元素沿着次轴（与主轴垂直的轴，比如若主轴方向为 row ，则次轴方向为 column ）的排列方式。可选项有：flex-start、center、flex-end 以及 stretch。

注意：要使 stretch 选项生效的话，子元素在次轴方向上不能有固定的尺寸。比如说，组件 style 中设置 flexDirection: column ，要使用 alignItems: stretch 生效，则子元素中的 width 不能指定固定值。

8 处理文件输入

TextInput 是一个允许用户输入文本的基础组件。它有一个名为 onChangeText 的属性，此属性接受一个函数，而此函数会在文本变化时被调用，另外还有一个名为 onSubmitEditing 的属性，会在文本被提交后（用户按下软件盘上的提交键）调用。

9 如何使用滚动视图

ScrollView 是一个通用的可滚动的容器，可以在其中放入多个组件和视图，通过 horizontal 属性设置滚动的方向。

ScrollView 适合用来显示数量不多的滚动元素。放置在 ScrollView 中的所有组件都会被渲染，哪怕有些组件因为内容太长被挤出了屏幕外。ListView 性能会更好。（类比到 iOS 中的 UIScrollView 和 UITableView）

10 如何使用长列表

RN 提供了几个适用展示长列表数据的组件，一般会选用 FlatList 或 SectionList 。

FlatList 并不立即渲染所有元素，而是优先渲染屏幕上可见的元素。

FlatList 组件必须的两个属性是 data 和 renderItem 。data 是列表的数据源，而 renderItem 则从数据源中逐个解析数据，然后返回一个设定好的组件来渲染。

SectionList 可以渲染一组需要分组的数据。属性是 sections ，renderItem 和renderSectionHeader 。

分别对应到 iOS 里的 UITableView 的 plain 和 group 类型。

使用其他的网络库

RN 中已经内置了 XMLHttpRequest API（即ajax）。一些基于 XMLHttpRequest 封装的第三方库也可以使用，例如 frisbee 或是 axios 等。但注意不能使用 jQuery，因为 jQuery 中还使用了很多浏览器中才有而 RN 中没有的东西。

WebSocket 支持

RN 还支持 WebSocket ，这种协议可以在单个 TCP 连接上提供全双工的通信信道。

关于 React Native 工程离线运行(详情看这里)

React Native 工程离线运行的重点是将 index.ios.bundle 文件保存在机器本地，index.ios.bundle 文件是工程编译后的 js 文件的打包，index.ios.bundle 将被 packager 服务器创建。

纯 RN 工程

纯 RN 工程创建好后在 Xcode 目录中就包含了 main.jsbundle 文件，只不过文件是不存在的（文件标红），这个文件就是编译的时候由 packager 服务器打包好的 index.ios.bundle 文件。

修改 jsCodeLocation

将初始化 URL 的代码

jsCodeLocation = [NSURL URLWithString:@"http://<ip地址>:8081/index.ios.bundle?platform=ios&dev=true"];

修改成

jsCodeLocation = [[NSBundle mainBundle] URLForResource:@"main" withExtension:@"jsbundle"];

打包

纯 RN 工程离线运行只按照正常的流程打包 app 即可，因为 index.ios.bundle 在打包过程中会自动生成并绑定到 main.jsbundle ，离线运行时就会根据 jsCodeLocation 找到 js 文件了。

混编工程

由于混编工程一开始是由 Xcode 创建的，并不存在 main.jsbundle 文件，因此需要手动创建。

生成本地 main.jsbundle

在项目根目录下执行

curl http://localhost:8081/index.ios.bundle -o main.jsbundle

如果 curl 命令执行失败，请确保服务器已开启，否则提示8081端口无法连接。

这里需要注意，执行这个命令的时候需要确保项目根目录存在 index.ios.js 文件。

执行成功后工程目录中就多了 main.jsbundle 文件，然后将他拖到 Xcode 中。

设置 jsCodeLocation 指向本地 main.jsbundle 文件

其他生成方式（参考这里）

除了上述的 curl 的方式生成，还可以使用 react-native bundle 命令生成。而且还在指定目录，并生成图片文件，不过在加入 Xcode 工程的时候 Assets 的资源勾选 Create folder reference。但是加入的 bundle 文件还是要参与编译，不然初始化的时候是找不到文件的。

然后，可以在 debug 的下指定初始化路径为 jsCodeLocation = [NSURL URLWithString:@"http://<ip地址>:8081/<js文件名>?platform=ios&dev=true"]; 这样就可以直接调试在 js 上修改的内容了，打包发布的时候注意需要重新打最新的 bundle 文件然后进行打包即可。
